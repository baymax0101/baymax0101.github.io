SQLAlchemy 2.0 + Alembic：数据库建模与版本迁移

FastAPI Python 数据库 SQLAlchemy Alembic
FastAPI 实战
发布日期:   2026-04-21

本文是《FastAPI 实战》系列第 3 篇。查看系列目录

前两篇我们搞定了路由和依赖注入，但一个正经的后端服务离不开数据库。在 Python 生态里，SQLAlchemy 几乎是 ORM 的事实标准，而 Alembic 则是它的官方迁移工具。但网上的 SQLAlchemy 教程大多是 1.x 的老写法，你照着抄很可能会踩坑——因为 2.0 的 API 变化非常大。

一句话摘要：从零搭建一套基于 SQLAlchemy 2.0 + Alembic 的异步数据库层，涵盖模型定义、关系映射、CRUD、迁移脚本和版本回滚，帮你避开新旧版本混用的坑。

背景与问题

假设你要做一个博客系统，有用户、文章、标签三张表。用户和文章是一对多关系（一个用户写多篇文章），文章和标签是多对多关系（一篇文章多个标签，一个标签下多篇文章）。

不用 ORM，你得手写 CREATE TABLE、INSERT、SELECT、JOIN，还得维护字段类型变更时的表结构同步。字段加一列、改个长度，都要手动写 ALTER TABLE 并在各个环境执行——这在大团队协作里简直是灾难。

ORM（Object-Relational Mapping） 的核心价值是：用 Python 类来定义表结构，用对象操作代替手写 SQL，同时让数据库 schema 的变更可追溯、可回滚。

SQLAlchemy 2.0 相比 1.x 有几个关键变化，也是很多人踩坑的原因：

变化点 1.x 写法 2.0 写法
查询语法 session.query(User).all() session.execute(select(User)).scalars().all()
异步支持 需要 asyncalchemy 等第三方包 原生支持 AsyncSession
模型基类 declarative_base() 返回一个基类 DeclarativeBase 作为基类，模型直接继承
字段定义 Column(Integer, primary_key=True) mapped_column(Integer, primary_key=True)
关系定义 relationship("Post", backref="author") relationship("Post", back_populates="author")

如果你在网上搜到的教程还在用 query() 和 Column()，那基本就是旧版——不要抄。

环境准备

你需要 Python 3.10+，安装以下依赖：

pip install sqlalchemy[asyncio] alembic asyncpg pydantic
  • sqlalchemy[asyncio]：SQLAlchemy 2.0，带异步扩展
  • alembic：官方迁移工具
  • asyncpg：PostgreSQL 的异步驱动（推荐生产用；本地测试可用 aiosqlite）
  • pydantic：用于数据校验（FastAPI 已自带）

本地快速测试用 SQLite：

pip install aiosqlite

实战代码

第一步：定义数据模型

我们先定义三张表，同时展示一对多和多对多关系。

# app/models.py
from datetime import datetime
from typing import List, Optional

from sqlalchemy import ForeignKey, Table, Column, String, Text, DateTime
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, relationship


class Base(DeclarativeBase):
    pass


# 多对多关联表
article_tag = Table(
    "article_tag",
    Base.metadata,
    Column("article_id", ForeignKey("articles.id"), primary_key=True),
    Column("tag_id", ForeignKey("tags.id"), primary_key=True),
)


class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(primary_key=True)
    username: Mapped[str] = mapped_column(String(50), unique=True, index=True)
    email: Mapped[str] = mapped_column(String(120), unique=True)
    created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow)

    # 一对多：一个用户有多篇文章
    articles: Mapped[List["Article"]] = relationship(
        back_populates="author", cascade="all, delete-orphan"
    )


class Article(Base):
    __tablename__ = "articles"

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(200))
    content: Mapped[Optional[str]] = mapped_column(Text, nullable=True)
    author_id: Mapped[int] = mapped_column(ForeignKey("users.id"))
    created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow)

    # 一对多反向引用
    author: Mapped["User"] = relationship(back_populates="articles")

    # 多对多：一篇文章多个标签
    tags: Mapped[List["Tag"]] = relationship(
        secondary=article_tag, back_populates="articles"
    )


class Tag(Base):
    __tablename__ = "tags"

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(50), unique=True, index=True)

    # 多对多反向引用
    articles: Mapped[List["Article"]] = relationship(
        secondary=article_tag, back_populates="tags"
    )

几个值得注意的点：

  • Mapped[T] 是 2.0 的类型注解写法，IDE 能自动补全，也能在 mypy 类型检查时报错。
  • mapped_column() 替代了旧版的 Column()。
  • relationship(back_populates=...) 替代了 backref，双向关系需要两边都写，虽然啰嗦但更明确。
  • cascade="all, delete-orphan" 表示删除用户时，他的文章也会被级联删除——谨慎使用，业务上可能不允许。

第二步：配置异步 Session

SQLAlchemy 2.0 原生支持异步，关键在于 create_async_engine + async_sessionmaker。

# app/db.py
import os
from contextlib import asynccontextmanager

from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession

DATABASE_URL = os.getenv("DATABASE_URL", "sqlite+aiosqlite:///./blog.db")

engine = create_async_engine(DATABASE_URL, echo=False)
AsyncSessionLocal = async_sessionmaker(
    engine, class_=AsyncSession, expire_on_commit=False
)


async def get_db() -> AsyncSession:
    """FastAPI Depends 用的 Session 生成器。"""
    async with AsyncSessionLocal() as session:
        try:
            yield session
        finally:
            await session.close()


@asynccontextmanager
async def get_db_context():
    """上下文管理器版本，适合脚本和测试。"""
    async with AsyncSessionLocal() as session:
        yield session

expire_on_commit=False 很重要：默认情况下 commit() 后对象会被”过期”，下次访问会触发懒加载。异步环境下懒加载会报错（因为对象绑定的是已关闭的 Session），关闭过期机制可以避免这个问题。

第三步：CRUD 操作

下面展示最常用的增删改查，以及关联查询。

# app/crud.py
from sqlalchemy import select
from sqlalchemy.orm import selectinload

from app.db import get_db_context
from app.models import User, Article, Tag


async def create_user(username: str, email: str) -> User:
    async with get_db_context() as db:
        user = User(username=username, email=email)
        db.add(user)
        await db.commit()
        await db.refresh(user)  # 获取自增 ID
        return user


async def get_user_with_articles(user_id: int) -> User | None:
    async with get_db_context() as db:
        result = await db.execute(
            select(User)
            .where(User.id == user_id)
            .options(selectinload(User.articles))
        )
        return result.scalar_one_or_none()


async def create_article(title: str, content: str, author_id: int, tag_names: list[str]) -> Article:
    async with get_db_context() as db:
        # 查找或创建标签
        tags = []
        for name in tag_names:
            result = await db.execute(select(Tag).where(Tag.name == name))
            tag = result.scalar_one_or_none()
            if tag is None:
                tag = Tag(name=name)
                db.add(tag)
            tags.append(tag)

        article = Article(
            title=title,
            content=content,
            author_id=author_id,
            tags=tags,
        )
        db.add(article)
        await db.commit()
        await db.refresh(article)
        return article


async def list_articles(skip: int = 0, limit: int = 10) -> list[Article]:
    async with get_db_context() as db:
        result = await db.execute(
            select(Article)
            .offset(skip)
            .limit(limit)
            .order_by(Article.created_at.desc())
        )
        return result.scalars().all()

关联查询需要特别注意：如果你 select(User) 后直接访问 user.articles，异步环境下会抛 MissingGreenlet 错误。解决办法是在查询时用 options(selectinload(User.articles)) 预加载关联数据。

第四步：在 FastAPI 中集成

# main.py
from fastapi import FastAPI, Depends, HTTPException
from sqlalchemy.ext.asyncio import AsyncSession

from app.db import get_db
from app.crud import create_user, get_user_with_articles
from app.schemas import UserCreate, UserResponse

app = FastAPI()


@app.post("/users", response_model=UserResponse, status_code=201)
async def register_user(payload: UserCreate, db: AsyncSession = Depends(get_db)):
    """注册新用户。"""
    user = await create_user(payload.username, payload.email)
    return user


@app.get("/users/{user_id}", response_model=UserResponse)
async def read_user(user_id: int, db: AsyncSession = Depends(get_db)):
    """获取用户信息及文章列表。"""
    user = await get_user_with_articles(user_id)
    if user is None:
        raise HTTPException(status_code=404, detail="User not found")
    return user

注意 create_user 内部用了 get_db_context()（自己开 Session），而接口层用 Depends(get_db)（FastAPI 注入 Session）。两者不要混用——否则会出现”Session 嵌套”的诡异问题。

第五步：Alembic 迁移

初始化 Alembic：

alembic init alembic

这会生成 alembic/ 目录和 alembic.ini。接下来修改两个关键文件。

修改 alembic/env.py，让它知道你的异步引擎：

# alembic/env.py
import asyncio
from sqlalchemy.ext.asyncio import create_async_engine

from app.models import Base  # 导入你的模型基类
from app.db import DATABASE_URL

config = context.config
config.set_main_option("sqlalchemy.url", DATABASE_URL)

target_metadata = Base.metadata


def run_migrations_offline():
    url = config.get_main_option("sqlalchemy.url")
    context.configure(
        url=url,
        target_metadata=target_metadata,
        literal_binds=True,
        dialect_opts={"paramstyle": "named"},
    )
    with context.begin_transaction():
        context.run_migrations()


async def run_migrations_online():
    connectable = create_async_engine(DATABASE_URL)
    async with connectable.connect() as connection:
        await connection.run_sync(do_run_migrations)
    await connectable.dispose()


def do_run_migrations(connection):
    context.configure(connection=connection, target_metadata=target_metadata)
    with context.begin_transaction():
        context.run_migrations()


if context.is_offline_mode():
    run_migrations_offline()
else:
    asyncio.run(run_migrations_online())

生成迁移脚本：

alembic revision --autogenerate -m "init_models"

这条命令会对比当前数据库 schema 和 Base.metadata，自动生成 CREATE TABLE 语句。

执行迁移：

alembic upgrade head

回滚到上一个版本：

alembic downgrade -1

回滚到特定版本：

alembic downgrade 8f3a2b1c4d5e  # revision ID

第六步：多环境配置

实际项目里，开发、测试、生产用的数据库地址不同。不要在 env.py 里硬编码，用环境变量：

# .env
DATABASE_URL=postgresql+asyncpg://user:pass@localhost/blog_dev

在 alembic.ini 中，把 sqlalchemy.url 留空或写占位符，然后在 env.py 中读取环境变量：

import os
DATABASE_URL = os.getenv("DATABASE_URL", "sqlite+aiosqlite:///./blog.db")
config.set_main_option("sqlalchemy.url", DATABASE_URL)

如果你需要同时支持异步驱动（asyncpg）和同步驱动（psycopg2），可以在 .env 里定义两个 URL，迁移时 Alembic 需要同步引擎，运行时用异步引擎。

踩坑记录

❌ 错误：混用 query() 和 execute()，或者旧版 Column() 和新版 mapped_column()

有些教程写 session.query(User).filter(...)，但 2.0 的异步 Session 没有 query() 方法。你应该用 session.execute(select(...))。

同样，Column(Integer, primary_key=True) 在 2.0 的 Mapped 类型系统下不会被类型检查器正确识别，必须用 mapped_column(Integer, primary_key=True)。

✅ 正确做法：全项目统一使用 SQLAlchemy 2.0 的 select() + execute() + mapped_column() + Mapped[T] 写法。

❌ 错误：异步环境下访问未预加载的关联属性

# 错误示范
user = await db.get(User, 1)
for article in user.articles:  # 这里会抛 MissingGreenlet！
    print(article.title)

异步引擎不支持懒加载（lazy loading），因为关联查询需要一次新的数据库请求，但原 Session 已经关闭了。

✅ 正确做法：查询时显式预加载关联数据。

result = await db.execute(
    select(User).options(selectinload(User.articles))
)
user = result.scalar_one()

或者如果你只需要一两个关联字段，用 joinedload() 做 JOIN 查询，比 selectinload() 的二次查询更高效。

❌ 错误：忘记 await db.commit()，以为 add() 会自动提交

# 错误示范
db.add(user)
# 没有 commit，数据根本没写进去！

SQLAlchemy 的 add() 只是把对象放入 Session 的”待处理队列”，真正的 INSERT 发生在 commit() 时。

✅ 正确做法：增删改后必须 await db.commit()，需要自增 ID 时再加 await db.refresh(obj)。

总结

SQLAlchemy 2.0 的 Mapped 类型系统和原生异步支持，让 Python ORM 终于跟上了现代类型安全编程的潮流。配合 Alembic，你可以用一套代码同时管理模型定义和 schema 迁移，团队协同时不再需要手动同步 ALTER TABLE。

下一步可以学什么：

  • 学习 Pydantic v2 的 model_validator 和 computed_field，在 schemas 层做更复杂的业务校验
  • 了解 数据库连接池 调优（pool_size、max_overflow），为高并发做准备
  • 为 CRUD 层写 pytest-asyncio 测试，用 aiosqlite 做内存数据库测试
